# main-portal-lab

## 项目简介

`main-portal-lab` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，而是作为一系列页面资源的集中存储与展示平台。每个页面可独立运行，适用于测试、演示、原型展示或快速分享静态内容。

## 目录结构

```
main-portal-lab/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面所需的静态资源（图片、样式、脚本等）
├── docs/           # 相关文档或说明
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件为一个独立页面，文件名即页面标识。
- `assets/`：按需组织，与页面文件对应，便于维护。
- `docs/`：可存放页面使用说明、变更记录或其他辅助文档。

## 页面归档说明

本仓库中归档的 HTML 页面均为独立自包含文件，或附带必要的资源文件。每个页面可单独打开运行，不需要额外服务器环境（除非页面本身依赖后端服务）。页面内容可能涉及各类实验、原型或静态展示，请以实际文件为准。

## 维护说明

- **添加页面**：将新的 HTML 文件放入 `pages/` 目录，如有配套资源请同步放入 `assets/` 对应子目录。
- **更新页面**：直接修改相应文件，建议在提交时注明变更内容。
- **删除页面**：移除对应文件及资源，并确保无其他引用。
- **版本记录**：建议在 `docs/` 下维护变更日志，或利用 Git 提交信息记录关键更新。

## 使用方式

1. 克隆或下载本仓库。
2. 在浏览器中打开 `pages/` 目录下的任意 HTML 文件即可查看对应页面。
3. 如需部署到 Web 服务器，将整个仓库内容上传至服务器根目录或子目录即可。

## 注意事项

- 本仓库仅用于页面归档与发布，不提供任何形式的服务或承诺。
- 页面内容可能随时更新或移除，请以仓库最新状态为准。
- 如有疑问或建议，欢迎通过 Issues 或 Pull Requests 参与维护。
